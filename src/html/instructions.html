Quick Start

Welcome! Follow these steps to get Angels-NowPlaying running with OBS and Tuna.

1. Install Angels-NowPlaying

Download the latest release from the GitHub Releases page and run the installer. Launch the app when installation is complete. (You should be here.)

2. Install and configure Tuna

  1. Install the Tuna plugin for OBS from the Tuna releases page. Special thanks to the Tuna project — this app wouldn't exist without their plugin.
  2. In OBS, go to Tools → Tuna and open the Web server tab.
  3. Enable the web server and note the port number (default is 1608).
  4. Connect Tuna to your music source — Spotify, YouTube Music Desktop App, Last.fm, a VLC Video Source in OBS, Windows Media Player, MPD, and others are all supported. See the Tuna documentation for source-specific setup steps.

3. Set the Tuna port in Angels-NowPlaying

Open Settings → Tuna Configuration in Angels-NowPlaying and confirm the port matches what Tuna is broadcasting on. Click Save Settings.

4. Pick and customise an overlay

From the home page, click Edit on any overlay card to open its editor. See Editor Basics for details on customising and saving.

OBS source size: The dimensions of the browser source for OBS listed in the editor next to the overlay name are suggested values for the default version of each overlay. If you have customised the overlay (resized elements, changed layout, etc.), you may need to adjust the browser source size in OBS to match your changes for the overlay to display correctly.

5. Add the overlay to OBS

  1. In OBS, add a Browser Source to your scene.
  2. Check Local File and paste the path you copied in step 4. (Or use the URL from Serve over HTTP if you've enabled that feature.)
  3. Set the Width and Height to match the size shown in the editor (displayed next to the overlay name).
  4. Click OK. Start playing music — track info, album art, and the progress bar should appear within a few seconds.
VLC Video Sources: If you use a VLC Video Source in OBS for local music playback, add it as a VLC Video Source (not a Media Source) — Tuna can only read track data from VLC Video Sources, not Media Sources. You can point it at an entire folder to play a playlist.

Editor Basics

Customising an overlay

  1. From the home page, click Edit on any overlay card to open its editor.
  2. Use the sliders and colour pickers to adjust colours, font sizes, spacing, and layout. The preview on the right shows the actual overlay running live — what you see there is exactly what OBS will render.
  3. When you're happy, click Save. The changes are written directly to the overlay's CSS file on disk — there is no download step.
  4. In OBS, right-click the Browser Source for that overlay and choose Refresh (or toggle the source off and back on) to pick up the new styles.

Copying the overlay file path

Each editor has a Copy Path button that puts the full path to that overlay's main.html on your clipboard, ready to paste into OBS as a Local File Browser Source. When Serve over HTTP is enabled the same button switches to Copy URL and copies the network URL instead.

Going back to edit later

You can return to any overlay's editor at any time. Just open Angels-NowPlaying, click Edit on the overlay, make your changes, and click Save. Your previous customisations are always loaded from the saved CSS file.

Serve Overlays over HTTP

What it does

Angels-NowPlaying can host all your overlays on a small built-in HTTP server, so OBS (or any other browser) can load them by URL instead of from a local file path. This is useful when:

  • You run OBS on a different machine than the one editing overlays.
  • You want a friend or co-host to use one of your overlays from their own PC on the same network.
  • You'd rather paste a URL into OBS than a long file path.

Turning it on

  1. Open Settings and find the Serve Overlays over HTTP card.
  2. Flip the toggle on. The status line will switch to Currently serving on http://… when the server starts.
  3. Click Configure… to choose the port (default 8253) or to enable access from other devices on your network.

The server starts automatically whenever the app launches if you've left the toggle on, so you only need to set it up once.

Local-only vs. network access

By default the server only listens on 127.0.0.1 — that means only OBS (and other apps) running on the same PC as Angels-NowPlaying can reach it. This is the safest setting and is fine for almost everyone.

If you turn on Allow access from other devices on my network in the Configure modal, the server starts listening on all network interfaces and any device on the same Wi-Fi/LAN can request your overlays at http://<your-ip>:<port>/<overlay-id>/main.html.

Security note: The server has no authentication. With LAN access on, anyone on your network can load any overlay you have installed — including custom ones. Only enable this on networks you trust (your home Wi-Fi, not a coffee-shop hotspot).

Getting the URL into OBS

  1. Open the editor for the overlay you want to use.
  2. Click the header button — when the server is running it reads Copy URL. Paste the result into OBS as a URL Browser Source (not Local File).
  3. If you want OBS on another PC to load it, make sure that PC is on the same network and the URL uses your machine's LAN IP (the Copy URL button picks the LAN address automatically when LAN access is on).

Overlay Management

Installing custom overlays

Go to Settings → Overlay Management → Install Overlay from Zip to install a custom overlay packaged as a zip file. Installed overlays appear on the home page alongside the built-in ones and can be removed from the same settings card.

Showing or hiding overlays

The toggles under Overlay Management control which overlays show up on the home page. You can hide built-in overlays you never use to keep the grid tidy, or hide your installed user overlays while you tweak them.

The AppData folder

The Open AppData button in Settings opens the folder where Angels-NowPlaying stores user overlays, fonts, settings, and backups. You don't normally need to touch anything there, but it's handy for troubleshooting or copying files between machines.

Updates & Backups

Auto-updates

Angels-NowPlaying checks GitHub Releases for new versions automatically when you open the Settings page (the result is cached for a few hours so it doesn't hammer the network). When an update is available the Check for Updates button changes to Update Available — clicking it downloads, verifies, and installs the new version, then restarts the app.

Updates are signed with an Ed25519 key, so the app refuses to install anything that wasn't published by the official release pipeline.

Backups

Before an auto-update is installed, the app captures a snapshot of your AppData folder (settings, user overlays, fonts) so a failed update can be rolled back. You can also create snapshots manually from Settings if you want one before making big changes.

Feedback & Support

If you find a bug, have a feature request, or want to contribute, please open an issue or pull request on GitHub. Include your OBS version, Tuna version, and a short description of how to reproduce the problem.

If you'd like to support development, the Buy Me a Coffee button at the top of this page is much appreciated. Every contribution goes back into improving the app.
